# V_PL Frontend Concept

Use this skill for local HTML/CSS/JS prototypes.

## Gate

Before building a UI concept, confirm:

- What workflow the screen demonstrates.
- What visual source or style direction it should follow.
- Whether the prototype should be fully interactive or mostly static.

## Concept Rules

- Build the actual operational surface, not a marketing landing page.
- Prioritize dense but readable operator workflows: live video, map, route,
  telemetry, timeline, search, sync controls, snapshots, and annotations.
- Use realistic synthetic data only.
- Make all first-screen controls functional enough to explain the workflow.
- Keep responsive layouts stable; text must not overflow buttons, panels, cards,
  or map/video overlays.

## Verification

Run `npm run verify`. For visible changes, open or screenshot the local HTML
before claiming the concept is ready.
