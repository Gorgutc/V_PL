# V_PL Agent Rules

This file is the source of truth for `Gorgutc/V_PL` for both OpenAI Codex and
Claude Code. Codex reads it natively; Claude Code imports it through
`CLAUDE.md` (`@AGENTS.md`).

The canonical skills, agents, and hooks live in the Codex layer:
`plugins/v-pl-codex/`, `.agents/skills/`, `.codex/`, and `docs/agent/`.
The Claude layer under `.claude/` is generated by `npm run sync:harness`.
Do not edit generated Claude mirrors by hand.

## Repository

- Product: V_PL, a video platform concept for RTSP ingest, product/time binding,
  video playback, map and telemetry synchronization, searchable video archive,
  selected frame/label storage, and later video analytics.
- Current state: local bootstrap repository. Treat user requirements as the
  domain source until a formal backend/frontend stack is chosen.
- Concept work: static HTML/CSS/JS prototypes are acceptable for local review
  when the user has approved the design brief and visual direction.
- Verification: `npm run verify`; success means the command exits cleanly and
  reports `0 FAIL`, without relying on historical pass totals.
- Publish flow: future shipped work should use a `codex/*` branch, be pushed to
  GitHub, and use a draft PR unless the user says otherwise.

## Authority

1. Current user request.
2. `verify-frozen.js` and its current passing baseline.
3. This `AGENTS.md`.
4. Repo-local plugin skills in `plugins/v-pl-codex/skills/`.
5. Supplemental agent docs in `docs/agent/`.
6. Supplemental project-local skills in `.agents/skills/`.
7. Supplemental agent and hook contracts in `.codex/`.
8. Generated Claude Code mirror in `.claude/skills/`, `.claude/agents/`, and
   `.claude/settings.json`.

If a generated mirror disagrees with the canonical Codex layer, the mirror is
stale. Update the canonical source and run `npm run sync:harness`.

## Domain Contracts

- Use mock or synthetic video, telemetry, routes, frames, and labels in local
  concepts unless the user explicitly provides approved real material.
- Do not commit credentials, RTSP URLs containing secrets, operational private
  video, personal files, or generated large binary captures.
- Bind every video concept to three primary dimensions: product, time, and
  geographic/route context when available.
- Keep search concepts aligned to area, date/time window, and product.
- Keep synchronous viewing concepts explicit about the shared clock that drives
  video, route, and telemetry panes.
- Treat video analytics as a separate layer from raw ingest/playback. Analytics
  may reference orthophoto creation, GeoServer viewing, Metashape integration,
  geolocation, change detection, and operator annotations, but concepts should
  avoid pretending those pipelines already exist.

## Working Rules

- Read before editing; use `rg` for file and text search.
- Keep edits scoped to the requested task and preserve unrelated user changes.
- Before UI prototype work, confirm the design brief and visual target required
  by Product Design workflow.
- Prefer repo-native instructions and executable checks over chat-only context.
- Put long-running decisions in `docs/agent/`; keep skill bodies short.
- Do not add dependencies or network-backed tooling without a clear need and
  user approval when sandbox escalation is required.

## Primary Skills

- `v-pl-rules`: repository source-of-truth rules and bootstrap context.
- `v-pl-video-domain`: ingest, storage, product/time binding, frame/label
  retention, and playback contracts.
- `v-pl-frontend-concept`: static HTML/CSS/JS concept work after the design
  brief and visual direction are approved.
- `v-pl-map-telemetry`: synchronized map, route, telemetry, and video review.
- `v-pl-video-analytics`: orthophoto, GeoServer, geolocation, change detection,
  and real-time operator annotation concepts.
- `v-pl-context-keeper`: narrow read-only repository context gathering.
- `v-pl-quality-gate`: review frontend/domain changes before handoff.
- `v-pl-audit-skills`: check instructions, skills, hooks, and mirrors for drift.
- `v-pl-ship`: final pre-commit/pre-push workflow.
